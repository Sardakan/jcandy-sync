# Микросервис синхронизации (МойСклад <-> Сайт)

Данный сервис обеспечивает двустороннюю синхронизацию данных между интернет-магазином и учетной системой МойСклад. Система построена на модульной архитектуре, оптимизирована для работы в Docker и учитывает лимиты API МойСклад.

## Основные возможности
- **Синхронизация товаров**: Перенос номенклатуры, цен, описаний и изображений.
- **Синхронизация заказов**: Автоматическое создание заказов в МойСклад при покупке на сайте.
- **Синхронизация контрагентов**: Создание и обновление карточек клиентов.
- **Очередь задач**: Гарантированная доставка данных. Если API временно недоступно, задача будет повторена позже.
- **Массовая миграция**: Специальные инструменты для первичного наполнения базы данных МойСклад.

## Структура проекта

### 1. `app.js` (Ядро и API)
Точка входа. Инициализирует Express-сервер. Содержит маршруты для приема данных с сайта и обработки вебхуков от МойСклад. Все тяжелые операции выполняются асинхронно через очередь.

### 2. `processor.js` (Бизнес-логика)
Центральный узел обработки данных. Здесь происходит маппинг (преобразование) полей сайта в поля МойСклад. 
- **Важно**: В текущей версии логика синхронизации остатков (`syncStocksFromDocument`) закомментирована по требованию заказчика.

### 3. `queue.js` (Очередь и отказоустойчивость)
Реализует надежное хранение задач в `queue.json`.
- **Retry Policy**: При ошибке задача возвращается в очередь.
- **DLQ (Dead Letter Queue)**: После 3 неудачных попыток задача переносится в `failed_queue.json` для ручного анализа, чтобы не блокировать основной поток.

### 4. `moysklad.js` & `siteApi.js` (Интеграции)
Низкоуровневые клиенты для работы с API. 
- `moysklad.js` автоматически обрабатывает ошибки формата JSON от МС и предоставляет методы для работы с метаданными и изображениями.
- `siteApi.js` отвечает за отправку обновлений обратно на сайт.

## Настройка (Файл .env)
Для работы сервиса необходимо создать файл `.env` в корневой директории:
```env
PORT=3000
MS_LOGIN=ваш_логин_мс
MS_PASSWORD=ваш_пароль_мс
SITE_API_URL=https://ваш-сайт.ru/api
SITE_API_TOKEN=ваш_токен
```

## Инструкция по эксплуатации

### Массовая миграция товаров
Для первичного переноса товаров с сайта в МойСклад используется эндпоинт:
`POST /api/v1/admin/mass-migrate-products`
Этот процесс разбит на батчи по 25 товаров с паузами в 15 секунд для соблюдения лимитов API МойСклад и экономии оперативной памяти.

### Работа с очередью
- **Просмотр логов**: Все действия записываются в `sync.log`.
- **Очистка очереди**: Если нужно сбросить все текущие задачи, используйте `POST /api/v1/admin/clear-queue`.
- **Ошибки**: Если синхронизация не проходит, проверьте `failed_queue.json`.

### Вебхуки
Сервер ожидает вебхуки от МойСклад по адресу `/api/v1/webhooks/ms`. 
Для корректной работы синхронизации "из МС на сайт" необходимо настроить вебхуки в личном кабинете МойСклад на этот URL.

## Запуск проекта

### Вариант 1: Обычный запуск (Node.js)
Этот способ подходит для тестирования на локальном компьютере.
1. Установите [Node.js](https://nodejs.org/) (рекомендуемая версия 18 или выше).
2. Откройте терминал в папке с проектом.
3. Установите зависимости (если есть файл package.json):
   ```bash
   npm install
   ```
4. Создайте файл `.env` и заполните его данными (см. раздел "Настройка").
5. Запустите сервис:
   ```bash
   node app.js
   ```

### Вариант 2: Запуск через Docker (Рекомендуется)
Docker позволяет запустить проект в изолированном контейнере, что исключает проблемы с версиями Node.js и зависимостями.

1. **Установите Docker**: Скачайте и установите [Docker Desktop](https://www.docker.com/products/docker-desktop/) для вашей ОС.
2. **Подготовьте .env**: Убедитесь, что файл `.env` находится в папке с проектом.
3. **Сборка образа**:
   Откройте терминал в папке проекта и выполните команду:
   ```bash
   docker build -t ms-sync .
   ```
   *Эта команда создаст "образ" вашего приложения под названием `ms-sync`.*
4. **Запуск контейнера**:
   Выполните команду:
   ```bash
   docker run -d --name my-sync-app --env-file .env -p 3000:3000 ms-sync
   ```
   *Что значат эти параметры:*
   - `-d`: Запустить в фоновом режиме.
   - `--name my-sync-app`: Название вашего запущенного приложения.
   - `--env-file .env`: Использовать настройки из вашего файла .env.
   - `-p 3000:3000`: Пробросить порт 3000 из контейнера на ваш компьютер.

5. **Как проверить, что всё работает?**
   - Введите в браузере `http://localhost:3000`. Если сервер запущен, вы увидите ответ (или ошибку 404, если корень не настроен, но главное — сервер ответит).
   - Посмотреть логи внутри Docker: `docker logs my-sync-app`
   - Остановить приложение: `docker stop my-sync-app`
