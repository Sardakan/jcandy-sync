# Микросервисы синхронизации (МойСклад <-> Сайт)

Этот каталог содержит модульную версию скрипта синхронизации (`sync.js`), разбитую на логические компоненты для удобства поддержки, масштабирования и деплоя в Docker.

## Структура проекта

### 1. `app.js` (Точка входа)
Главный файл приложения. Содержит инициализацию Express-сервера и все маршруты (роуты) API. 
Здесь нет бизнес-логики, только прием запросов, валидация базовых параметров и передача задач в очередь или напрямую в обработчики.

### 2. `config.js` (Конфигурация)
Централизованное хранилище всех настроек. 
Читает переменные окружения из файла `.env` (или `process.env` в Docker). Содержит базовые URL, пути к файлам очередей и логов, а также константы (например, `ORDER_STATES` для маппинга статусов заказов).

### 3. `logger.js` (Логирование)
Модуль для асинхронной записи логов. 
Использует `fs.promises.appendFile`, чтобы не блокировать Event Loop (главный поток Node.js) при записи на диск.

### 4. `queue.js` (Управление очередью)
Реализация асинхронной очереди задач (`SyncQueue`).
- Сохраняет задачи в `queue.json`.
- Защищает файл от повреждений при одновременной записи (Race Condition) с помощью флага `isSaving`.
- Реализует паттерн **Dead Letter Queue (DLQ)**: если задача завершается с ошибкой более 3 раз, она переносится в `failed_queue.json`, чтобы не блокировать обработку новых задач.

### 5. `siteApi.js` (Клиент API сайта)
Модуль для взаимодействия с базой данных/API вашего сайта.
Использует преднастроенный экземпляр `axios` (`siteApiClient`) с базовым URL и токеном авторизации. Содержит функцию `siteRequest` для выполнения запросов.

### 6. `moysklad.js` (Клиент API МоегоСклада)
Модуль для взаимодействия с API МоегоСклада.
- Содержит преднастроенный `axios` (`msApiClient`).
- Реализует кэширование метаданных (`cache`), чтобы не делать лишних запросов.
- Содержит методы для поиска товаров по штрихкоду, создания атрибутов, загрузки изображений в Base64 и работы с остатками.
- Перехватывает и логирует детальные ошибки от МоегоСклада (массив `errors`).

### 7. `processor.js` (Бизнес-логика)
Самый важный модуль. Содержит объект `syncProcessor`, который берет задачи из очереди и выполняет фактическую синхронизацию:
- `syncProduct`: Создание или обновление товара в МС (включая загрузку картинок с обработкой ошибок).
- `syncOrder`: Создание заказа в МС (с привязкой контрагента и позиций).
- `syncCounterparty`: Создание или обновление контрагента в МС.

## API Эндпоинты (Инструкция для Postman)

### 1. Синхронизация из БД сайта в МойСклад (PUSH)
Используется для ручного или автоматического добавления сущностей в очередь синхронизации.
* `POST /api/v1/external/products` — Добавить товар в очередь (в теле JSON товара)
* `PATCH /api/v1/external/products/:barcode` — Обновить товар в очереди
* `POST /api/v1/external/orders` — Добавить заказ в очередь
* `POST /api/v1/external/counterparties` — Добавить контрагента в очередь

### 2. Массовая миграция (Тестовые данные)
Запрашивает список предопределенных ID с сайта и ставит их в очередь.
* `POST https://jcandy-sync.onrender.com/api/v1/admin/migrate` — Миграция тестовых товаров
* `POST https://jcandy-sync.onrender.com/api/v1/admin/migrate-orders` — Миграция тестовых заказов
* `POST https://jcandy-sync.onrender.com/api/v1/admin/migrate-counterparties` — Миграция тестовых контрагентов

### 3. Проверка данных в МойСклад (FETCH)
Прямые запросы в API МойСклад для проверки текущего состояния.
* `GET https://jcandy-sync.onrender.com/api/v1/external/products/:barcode` — Посмотреть товар в МС по штрихкоду
* `GET https://jcandy-sync.onrender.com/api/v1/external/orders/:id` — Посмотреть заказ в МС по номеру (name)
* `GET https://jcandy-sync.onrender.com/api/v1/external/counterparties/:email` — Посмотреть контрагента в МС по Email

### 4. Управление очередью и Вебхуки
* `POST https://jcandy-sync.onrender.com/api/v1/admin/clear-queue` — Очистить очередь задач (`queue.json`)
* `POST https://jcandy-sync.onrender.com/api/v1/webhooks/ms` — Прием вебхуков от МойСклад (обновление статусов)
