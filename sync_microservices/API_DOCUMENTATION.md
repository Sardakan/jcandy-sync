# Документация API микросервиса синхронизации (JCandy <-> МойСклад)

Этот документ описывает эндпоинты, которые бэкенд сайта должен использовать для передачи данных в МойСклад.

## Базовая информация
- **Base URL:** `https://jcandy-sync.onrender.com`
- **Формат данных:** JSON
- **Заголовки:** `Content-Type: application/json`
- **Логика работы:** Все запросы на запись (POST/PATCH) являются асинхронными. Они ставятся в очередь и обрабатываются с задержкой в 3 секунды для соблюдения лимитов API МойСклад.

---

## 1. Товары (Products)
Используются для синхронизации каталога. Основной ключ — `barcode`.

### Создать/Добавить в очередь
`POST /api/v1/external/products`
- **Тело запроса:** Объект товара.
- **Обязательные поля:** `barcode`, `title`.

### Массовое обновление
`PATCH /api/v1/external/products/bulk`
- **Тело запроса:** Массив объектов товаров или объект с ключом `items`.

### Обновить конкретный товар
`PATCH /api/v1/external/products/:barcode`
- **URL параметр:** `:barcode` — штрихкод товара.
- **Тело запроса:** Поля, которые нужно обновить.

### Получить данные из МойСклад
`GET /api/v1/external/products/:barcode`
- Возвращает текущее состояние товара напрямую из API МойСклад.

---

## 2. Заказы (Orders)
Используются для передачи заказов. Основной ключ — `id` (номер заказа).

### Создать заказ
`POST /api/v1/external/orders`
- **Тело запроса:** Объект заказа.
- **Особенности:** Если в заказе есть товар, которого нет в МойСклад, сервис автоматически попытается создать его, запросив данные с сайта.

### Массовое обновление заказов
`PATCH /api/v1/external/orders/bulk`
- **Тело запроса:** Массив объектов заказов.

### Обновить заказ
`PATCH /api/v1/external/orders/:id`
- **URL параметр:** `:id` — номер заказа.

### Получить данные заказа
`GET /api/v1/external/orders/:id`
- Возвращает данные заказа и его текущий статус в МойСклад.

---

## 3. Контрагенты (Counterparties)
Используются для синхронизации покупателей. Основной ключ — `email`.

### Создать покупателя
`POST /api/v1/external/counterparties`
- **Тело запроса:** Объект покупателя.
- **Обязательные поля:** `email`.

### Массовое обновление покупателей
`PATCH /api/v1/external/counterparties/bulk`
- **Тело запроса:** Массив объектов покупателей.

### Обновить покупателя
`PATCH /api/v1/external/counterparties/:email`
- **URL параметр:** `:email` — Email покупателя.

### Проверить покупателя
`GET /api/v1/external/counterparties/:email`
- Возвращает данные контрагента из МойСклад.

---

## 4. Служебные эндпоинты

### Вебхук МойСклад
`POST /api/v1/webhooks/ms`
- Предназначен для настройки в кабинете МойСклад.
- **Функции:** 
  1. Обновляет статусы заказов на сайте при их изменении в МС.
  2. Массово обновляет остатки товаров на сайте (`Остаток - Резерв`) при любых изменениях в документах МС.

### Очистка очереди
`POST /api/v1/admin/clear-queue`
- Полностью очищает список ожидающих задач.

---

## Формат ответов
Для всех POST/PATCH запросов сервис возвращает подтверждение постановки в очередь:
```json
{
  "status": "queued",
  "entity": "product",
  "id": "123456789"
}
```
